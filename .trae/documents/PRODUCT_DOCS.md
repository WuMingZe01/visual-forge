# Visual Forge — 产品文档

> AI 驱动的电商视觉物料工作台 — 主图生成 / 详情页 / 批量工单 / 通用生图 / 提示词管理 / 任务历史

---

## 1. 系统概述

Visual Forge 是一个面向电商运营人员的 AI 图像生成工作台，通过对接 **领猫 SCM** 获取商品资料、调用 **Kimi/DeepSeek** 等 LLM 自动生成提示词、再通过 **Grsai/Yunwu** 图像引擎生成主图和详情页素材。

### 1.1 技术栈

| 层 | 技术 |
|---|------|
| 前端 | React + TypeScript + Vite + Tailwind CSS + Zustand |
| 代理 | Vite Proxy（解决 CORS） |
| LLM | Moonshot Kimi K2.6（多模态）+ DeepSeek V4 Flash（文本） |
| 图像引擎 | Grsai gpt-image-2 + Yunwu /images/generations |
| 数据 | 领猫 SCM API + localStorage 持久化 |
| 主题 | CSS 变量实现 5 套皮肤动态切换 |

### 1.2 路由表

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 款式管理 | 从领猫批量拉取款式，展示完整扩展信息 |
| `/studio` | 主图 | AI 换衣主图生成（双轨提示词） |
| `/batch` | 批量工单 | 批量输入款号 → 自动拉参数 → 逐行确认生成 |
| `/canvas` | 详情页 | 独立生成电商详情页模块 |
| `/general` | 通用生图 | 风格库 + 自由提示词生图 |
| `/history` | 任务历史 | 全部生图任务管理，支持批量下载 |
| `/prompts` | 提示词模板 | 内置模板库 + 新建/编辑/复制 |
| `/settings` | 系统设置 | API 密钥 / LLM 配置 / 领猫配置 / 默认提示词 |

## 2. 核心功能

### 2.1 款式管理 (`/`)

**数据来源**：领猫 SCM 系统，通过 `/lingmao-api` 代理到 `open.scm321.com`

**功能**：
- **批量导入**：从 Excel 复制款号列粘贴，一键拉取全部商品信息
- **本地款式库**：拉取后存入 localStorage，支持搜索、查看详情、删除
- **完整扩展信息**：每个款展示 30+ 字段，分为基础标识 / 规格合规 / 面料版型 / 包装物流 / 其他信息 5 个分类区
- **进入试衣**：选中款式直接跳转主图页面并加载产品资料

### 2.2 主图 (`/studio`)

**两种工作模式**：

| 模式 | 说明 |
|------|------|
| 单图生成 | 上传商品白底图 + 模特参考图 → 生成换衣主图 |
| 姿势裂变 | 多槽位独立参考图 + 姿势描述 → 批量生成多姿势图 |

**提示词生成（双轨制）**：

```
┌────────────────────────────────────────────┐
│  业务人员层面（中文方案，可修改）                │
├────────────────────────────────────────────┤
│  LLM 生成的中文三步方案：                      │
│  ① 生图方案概述  ② 详细提示词  ③ 负面提示词      │
├────────────────────────────────────────────┤
│  API 层面（英文提示词，自动生成，不可见但可预览）  │
│  Virtual try-on: fashion model wearing...   │
│  + 选款时自动注入产品资料                       │
│  + 保持不变特征 / 光影 / 构图                    │
│  最终发送给 Grsai GPT-Image-2                 │
└────────────────────────────────────────────┘
```

**AI 智能生成管线（两阶段）**：

| 步骤 | LLM | 输入 | 输出 |
|------|-----|------|------|
| Step 1 | Kimi K2.6（多模态） | 模特参考图 base64 | 5 类不变特征（中文） |
| Step 2 | DeepSeek V4 Flash | 不变特征 + 领猫产品资料 | 中文分步生图方案 |

**图片处理**：
- 上传支持 JPG/PNG/WebP，最大 50MB
- 发送前自动 Canvas 压缩到 2000px 长边 + JPEG 92% 质量，平衡速度和 AI 识别精度

**生成引擎路由**：
- `gpt-image-2` / `gpt-image-2-vip` → POST `/grsai/v1/api/generate`
- `nano-banana*` → POST `/grsai/v1/draw/nano-banana`
- 其他 → POST `/yunwu/v1/images/generations`

### 2.3 批量工单 (`/batch`)

**流程**：
1. 从 Excel 粘贴款号列 → 解析成行
2. 系统自动查领猫获取每款产品参数
3. 逐行上传模特图 + 商品图，可修改提示词、模型、分辨率
4. 一键批量生成，结果同步写入任务历史

### 2.4 详情页 (`/canvas`)

- 独立运行，无需先生成主图
- 支持手动上传参考图
- 9 个预设模块（主图 / 细节 / 面料 / 尺码 / 卖点 等）
- 每个模块独立编辑标题和提示词，独立生成
- 9:16 竖版拼接预览

### 2.5 通用生图 (`/general`)

- 3 个 Tab：自由生图 / 风格库（28 种预设）/ 历史记录
- 模型选择、比例、提示词翻译、参考图上传

### 2.6 任务历史 (`/history`)

- 所有生图任务（主图 / 详情页 / 通用）自动记录
- 入参可见：提示词、引擎、模型、分辨率
- 支持单张下载、批量下载、多选下载
- 按状态筛选：全部 / 生成中 / 已完成 / 失败

### 2.7 提示词模板库 (`/prompts`)

- 4 个内置模板（2 主图 + 2 详情页）
- 新建/编辑/复制/删除
- 分类筛选：全部 / 主图 / 详情页

### 2.8 系统设置 (`/settings`)

| 区块 | 内容 |
|------|------|
| AI 大语言模型 | 多模态模型 + 文本模型分别配置，支持启用/禁用/测试连接 |
| 生图引擎 API | Yunwu API Key + Grsai API Key + OSS（可选） |
| 默认提示词 | 主图和详情页的默认提示词模板 |
| 领猫 SCM | Base URL / 款式查询路径 / App ID / App Secret |
| 自定义图像模型 | 添加自定义 gpt-image-2 变体 |

## 3. 数据流

```
用户操作 → React 组件 → Zustand Store → API Service → Vite Proxy → 外部API
                                                                  ↓
                                                         Grsai / Yunwu / Kimi / DeepSeek / 领猫
                                                                  ↓
                                                               返回结果 → Zustand → 组件更新
                                                                  ↓
                                                         任务记录 → localStorage（持久化）
```

### 3.1 关键 Store

| Store | 职责 |
|-------|------|
| `useAppStore` | 系统配置（API Key / 引擎 URL） |
| `useTryOnStore` | 主图/详情页状态（图片 / 参数 / 结果） |
| `useLlmStore` | LLM 配置（多模态 + 文本模型）|
| `useTaskHistoryStore` | 任务历史记录和下载 |
| `useThemeStore` | 5 套皮肤切换 |

### 3.2 代理配置

| 路径前缀 | 目标 |
|---------|------|
| `/grsai` | `grsai.dakka.com.cn` — 图像生成 |
| `/yunwu` | `yunwu.ai` — 图像生成备用 |
| `/llm-deepseek` | `api.deepseek.com` — 文本 LLM |
| `/llm-moonshot` | `api.moonshot.cn` — 多模态 LLM |
| `/lingmao-api` | `open.scm321.com` — 领猫 SCM |

所有代理设置 `timeout: 300000`（5 分钟），`changeOrigin: true`。

## 5. 本地开发

```bash
cd web
npm install
npm run dev        # http://localhost:5177
npx tsc --noEmit   # 类型检查
```

### 5.1 配置 LLM

1. 打开 `/settings`
2. 在「AI 大语言模型配置」区域确认多模态和文本模型已启用
3. 点击「测试」验证连接
4. 到 `/studio` 使用「AI 智能生成方案」

### 5.2 配置生图引擎

在 `/settings` →「生图引擎 API 配置」填写 Grsai 和 Yunwu 的 API Key。

### 5.3 测试图片

`测试图片/` 目录提供三张测试图片：
- `BM26A050CM-正面.png` — 商品正面白底图
- `BM26A050CM-背面.png` — 商品反面白底图
- `model.jpg` — 模特参考图
